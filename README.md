## О проекте
Проект представляет собой **систему учета сотрудников и их должностей в организации**.

### Используемая архитектура
В качестве архитектуры используется язык программирования Python, веб-фреймворк Flask, система API Postman и система база данных SQLite.

### Цели проекта
Основными целями проекта являются *централизация* и безопасное *хранение кадровых данных*, *автоматизация процессов* обновления информации, *обеспечение контролируемого доступа* к конфиденциальным данным в зависимости от должностных обязанностей, а также упрощение *формирования отчетности* для руководства.

### Пользователи в системе
Пользователями выступают работники организации с различными уровнями доступа, основные роли в системе:
1. Роль *Сотрудник* позволяет только просматривать свой профиль.
2. Роль *HR* имеет доступ на чтение и запись данных сотрудников, включая регистрацию новых сотрудников.
3. Роль *Секретарь* позволяет просматривать данные сотрудников и создавать отчеты.

### Основыне функции системы
- авторизация пользователей по ролям
- многоуровневое отображение данных в зависимости от прав доступа
- редактирование профилей сотрудников
- ведение истории изменений данных
